    简介¶

    本文档主是pthread的基础使用方法

    头文件¶

    #include <pthread>
     

    线程变量和相关函数¶

    声明一个线程变量(线程ID)¶

    pthread_t th;   // 声明一个线程变量(线程ID)

    获取线程ID¶

    pthread_t pthread_self(void); // 获取当前线程ID

    比较是否为同一线程¶

    int pthread_equal(pthread_t tid1, pthread_t tid2); // 比较tid1和tid2是否为同一个线程
     

    线程函数声明模板¶

     void* myfunc(void* args) { }
     

    创建一个线程¶

    参数 作用
    thread 线程变量
    attr 未知
    start_routine 线程函数
    arg 线程传入参数 
    int pthread_create(pthread_t *thread, const pthread_attr_t *attr, void *(*start_routine) (void *), void *arg);

    example¶

    struct 
{
    /* add your arg list in this place */
}MY_ARG;

pthread_t th;
MY_ARG arg;

pthread_create(&th, NULL, myfunc, &arg);  //创建一个线程 并运行
     

    等待线程完成¶

    pthread_join(th, NULL);  // 等待该线程完成

    该操用与可结合的线程 下部分有讲  

    线程假共享¶

    线程假共享 又称 Fale Share¶

      由于CPU的多核操作，一个线程由一个核心完成，每一个核心都有独自的缓存区( cahe )，CPU会拷贝一部分内存到 cahe 中当修改完值后，需要和原内存比较，若该段内存部分修改则需先重新拷贝到 cahe 中在将 cahe 中的值拷贝到内存中
      当两个线程使用内存块相近时， 一个CPU核心将拷贝的内容包含另一个CPU核心的内容，导致内存反复拷贝内存到 cahe 中，消耗时间。  

    互斥量¶

    存在意义¶

      当两个线程使用同一个资源时，由于使用非原子操作，导致出现意料之外的结果，使用互斥量可以避免此问题。

      注意：互斥量只锁代码，不锁变量

    线程类别¶

    可结合的线程 joinable
    分离的线程 detached

    原子操作¶

    定义¶

      如果这个操作所处的层(layer)的更高层不能发现其内部实现与结构，那么这个操作是一个原子(atomic)操作。
      原子操作可以是一个步骤，也可以是多个操作步骤，但是其顺序不可以被打乱，也不可以被切割而只执行其中的一部分。 将整个操作视作一个整体是原子性的核心特征
